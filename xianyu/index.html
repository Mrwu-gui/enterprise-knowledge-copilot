为什么需要这套系统

企业在用大模型时的真实痛点

不是"能不能回答"的问题，而是怎么在真实业务里用起来

01

一个总机器人打天下

无法按业务拆分角色，所有问题塞给一个助手，回答混乱不可控

02

知识库杂乱难维护

文档越来越多，没有分类、标签、切片管理，后期维护成本极高

03

只能问答，不能执行

只能做简单对话，无法把业务流程自动化，无法调用企业内部接口

04

接不进企业内部系统

无法对接 ERP、CRM、OA 等业务系统，AI 变成了信息孤岛

05

出了问题无从排查

没有日志、没有调试、没有评测，效果好不好全靠感觉

06

私有化交付困难

后台不完整，客户接手难，交付质量全靠开发个人能力

核心能力

不是功能堆砌，是解决真实问题的能力组合

01

多智能体

一个租户配置多个智能体，每个独立设定：角色定位、Prompt、模型、知识范围、工具、MCP 服务、工作流。不同业务拆成不同助手，而非混用一个总机器人。

独立Prompt独立知识独立模型独立工作流
02

可视化工作流

14 种节点自由组合：AI 生成、知识检索、条件判断、HTTP 请求、脚本执行、MCP 调用、循环、并行、子流程、通知推送、延时、表单…把"问答"扩展成"业务执行"。

条件分支并行处理子流程复用HTTP调用
03

多租户隔离

一套系统服务多个企业客户。每个租户独立后台、品牌、知识库、模型配置、工作流。品牌+数据+配置三重隔离，适合 SaaS 运营和项目持续交付。

品牌隔离数据隔离配置隔离
04

结构化知识管理

知识库 → 分类 → 标签 → 文件，四级结构适合企业长期维护。支持多格式上传、网页导入、切片预览、按标签精细筛选。

多知识库标签管理切片预览网页导入
05

完整 RAG 检索链路

文档解析 → 文本抽取 → 切片 → 向量检索+关键词检索 → 混合召回 → Rerank 精排 → Query 改写。远超"简单喂文档给模型"的效果。

混合检索Rerank重排Query改写
06

MCP 与工具接入

后台配置工具和 MCP 服务，智能体与工作流均可调用。查业务数据、调内部接口、执行操作、读外部系统 — AI 不只是"回答"，还能"做事"。

MCP协议工具调用系统打通
07

图文问答

支持纯文本、图片上传、文字+图片混合提问。模型自动识别类型，不支持图文的模型自动限制上传入口。适合报告解读、票据识别、表单分析。

图片上传自动识别报告解读
08

运行观测与统计

聊天日志、请求日志、护栏事件、评测记录、调度记录、爬虫记录、租户/平台统计报表。交付后持续优化有据可依。

日志追踪效果评测统计报表
09

三端齐全

平台后台（服务商版）+ 租户后台 + 前台聊天页。完整后台体系，不是只有一个对话页面，可直接交付，客户接手无障碍。

平台后台租户后台前台聊天
