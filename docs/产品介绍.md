# 企业级 AI 知识库与 Agent 平台

## 一、产品简介

本项目是一套面向企业知识场景的 AI Agent 平台，支持企业私有知识接入、智能检索问答、自动化采集、租户隔离与后台配置管理。

它不是单纯的聊天壳，也不是仅供学习的 RAG Demo，而是一套可用于企业内部知识管理、制度问答、SOP 辅助、资料检索与多客户运营的产品型系统。

适用场景包括：

- 企业知识库问答
- 制度 / SOP / 操作手册检索
- 产品资料、FAQ、培训资料统一问答
- 多企业客户托管的 AI 知识服务
- 私有化部署的企业内部智能助手

---

## 二、产品核心能力

### 1. 多格式知识接入

系统支持企业常见知识文件直接上传入库，并统一进入检索链：

- Markdown
- TXT
- HTML
- PDF
- Word
- PowerPoint
- Excel
- 图片

文档处理链支持：

- 标题与章节识别
- 表格抽取
- OCR 文本识别
- 跨页内容修复
- 结构化切块

这意味着企业可以直接使用已有资料，而不需要手工重写成特殊格式。

### 2. 三层知识库架构

系统内置三层知识架构，用于区分稳定知识、阶段知识与高时效知识：

- **L1 基础库**：长期稳定、基本不变的知识
- **L2 增量库**：阶段性更新、偶尔变更的知识
- **L3 热库**：高时效、随时变化的知识

系统检索时会同时检索三层知识，并按默认权重进行优先级控制：

- `L3 > L2 > L1`

这种方式兼顾了：

- 最新信息优先
- 阶段性内容补充
- 长期规则兜底

### 3. 智能检索与知识调度

系统不是简单的“向量搜索 + 大模型问答”，而是包含完整的企业知识检索链：

- 向量检索
- BM25 稀疏检索
- 混合召回（Hybrid Retrieval）
- Rerank 重排序
- Query Rewrite
- 检索质量评估
- 自动重试与策略切换

系统会根据问题类型动态调整检索策略，例如：

- 编号 / 术语 / 精确字段类问题，更偏关键词检索
- FAQ / 解释说明类问题，更偏语义检索
- 制度 / 流程类问题，采用混合检索策略

### 4. 短期记忆能力

系统支持会话级短期记忆，用于提升连续对话体验：

- 记住最近几轮上下文
- 保留当前问题背景
- 避免多轮对话中上下文丢失

这使系统更接近“企业智能助手”，而不是每次都从零开始回答的机器人。

### 5. 企业级后台能力

系统提供完整后台配置能力，用于支持交付与运营：

- 知识库管理
- Python 采集脚本配置
- 模型配置
- 检索配置
- 企业定制化
- 问答测试
- 日志查看

### 6. 可观测性与安全护栏

系统内置运行日志与安全机制：

- 请求日志
- 聊天日志
- 护栏事件日志
- 调度执行历史
- 输入拦截
- Prompt 注入拦截
- 输出脱敏

这有助于企业更好地做：

- 运行监控
- 审计追踪
- 风险控制

## 四、技术架构

### 1. 服务层

- **FastAPI**

用于：

- API 服务
- 页面入口
- 后台接口
- 文件上传与版本导出

### 2. 编排层

- **LangGraph**

用于组织问答主链路，包括：

- 输入护栏
- 短期记忆
- 检索
- Prompt 组装
- 模型路由
- 回答生成
- 输出审计
- 日志与缓存写入

### 3. 文档与检索组件层

- **LangChain**

用于：

- DocumentLoader
- TextSplitter
- Retriever 适配
- 检索压缩组件

### 4. 向量数据库

- **Qdrant**

支持：

- 本地嵌入式模式
- 云端服务模式

### 5. Embedding / Rerank

支持多种可切换方案：

- SiliconCloud
- 百度千帆（Embedding）
- OpenAI Compatible
- 本地 fallback

### 6. 数据与缓存

- SQLite：业务数据存储
- Redis：语义缓存（支持回退到内存）

## 五、技术亮点

### 1. 多格式企业文档处理

支持从常见企业文件中直接提取知识，不依赖单一 Markdown 格式。

### 2. 三层知识调度

通过 L1 / L2 / L3 分层，平衡稳定规则、阶段资料与最新热知识。

### 3. 混合检索

结合：

- 语义向量检索
- BM25 稀疏检索

### 4. 模型 Key 池与自动降级

系统支持：

- 多 API Key 组成的 Key 池
- 主模型 / 备用模型双路配置
- Key 失效、限流或异常时自动切换
- 主模型失败时自动降级到备用模型

适合需要更高可用性的企业场景。
- Rerank 重排

提升企业文档中的检索精度。

### 4. 检索纠错机制

系统支持：

- Query Rewrite
- 检索质量评估
- 自动重试
- 路由切换

降低低质量召回导致的误答和幻觉。

### 5. 多租户隔离

服务商版中，每个企业具备独立的：

- 知识库目录
- Qdrant collection
- 模型配置
- 检索配置
- Python 脚本配置
- 日志与会话数据
