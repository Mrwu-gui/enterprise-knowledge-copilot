# 版本打包与交付说明

## 1. 适用场景

本文件用于说明如何从母版项目导出可销售版本。

当前只保留两种版本：

- `enterprise`：企业版（单后台）
- `service_provider`：服务商版（双后台）

这样做的目的不是把代码拆成多套，而是：

- 保持一套母版代码易维护
- 对外按版本切换入口与文档
- 让你可以快速打包、快速交付、快速售卖

## 2. 两种版本的区别

### 企业版

适合：

- 个人
- 小团队
- 单企业私有部署

保留：

- 单后台
- 账户管理
- 知识库
- Python 脚本
- 模型配置
- 企业定制化
- 日志查看
- 问答测试
- 前台登录与聊天

隐藏：

- 平台总后台
- 平台企业账户管理页
- 平台总日志页

### 服务商版

适合：

- 服务商
- 多客户托管
- SaaS 平台运营

保留：

- 平台总后台
- 租户后台
- 前台登录与聊天

## 3. 导出接口

### 查看版本

`GET /api/admin/release-profiles`

### 导出版本

`POST /api/admin/release/export`

请求体示例：

```json
{
  "profile_key": "enterprise"
}
```

### 下载导出包

`GET /api/admin/release/download?zip_name=...`

## 4. 打包中心

母版项目现在新增了一个只给你自己看的页面：

- `/factory`

这个页面不应该进入任何卖出的版本菜单，只用于：

- 读取当前可销售版本
- 选择企业版或服务商版
- 一键生成 zip 打包文件

## 5. 导出包内容

导出后的 zip 默认包含：

- 当前项目代码
- 版本化后的 `data/app_config.json`
- `docs/项目总说明.md`
- `docs/租户手册.md`
- 自动生成的 `RELEASE_EDITION.md`

## 6. 当前策略说明

当前导出采用轻量版本化，而不是深度裁代码：

- 保留共用代码底座
- 通过配置切换 `edition`
- 通过配置切换 `deployment_mode`
- 通过路由切换单后台 / 双后台入口

这样做的好处是：

- 实现简单
- 易维护
- 不会因为代码裁剪过深导致版本包难以维护

## 7. 当前推荐交付方式

### 卖企业版

对外强调：

- 单后台
- 企业账户管理
- 企业知识库
- 模型配置
- 自动采集
- 日志查看
- 问答测试

### 卖服务商版

对外强调：

- 双后台
- 多企业隔离
- 平台总后台
- 租户后台
- SaaS 运营能力
