# 使用教程

## 推荐使用顺序

1. 先配品牌和业务信息
2. 再配模型、检索、工具
3. 再导入知识
4. 再创建智能体和工作流
5. 最后做前台问答测试

## 平台后台怎么用

这一段只对服务商版有意义。

- 开通租户：新增企业客户、生成租户后台账号
- 看平台日志：统一查看整体调用和异常
- 做租户运营：统一维护多客户资料和交付状态

## 租户后台怎么用

- 账户管理：维护后台账号和成员信息
- 企业定制化：改名称、欢迎语、推荐问题、品牌文案
- 工作流配置：做默认流程、草稿、发布、试运行
- 知识库设置：管知识库、分类、标签、文件和知识范围
- Python 脚本设置：放辅助脚本或扩展逻辑
- 模型配置：配大模型、Embedding、Rerank、检索参数
- 日志查看：回看聊天日志、请求日志、护栏事件、评测和采集
- 问答测试：用真实问题做效果回归

## 前台聊天怎么用

- 用户从 `/login` 登录后进入 `/chat`
- 如果绑定多个智能体，可以切换不同助手
- 每个问题会自动走当前智能体绑定的知识、工作流和工具

## 最常见的实际动作

- 新增一个助手：建智能体、绑工作流、圈知识范围
- 新增一批资料：上传文件、补标签分类、验证切片和命中
- 调效果：先看日志和检索命中，再改 Prompt、模型或流程
- 做客户交付：先完成品牌、欢迎语、推荐问题和默认流程
