# 功能介绍

## 这是什么

这不是一个简单的“上传文档然后提问”的 Demo，而是一套可以持续交付、持续运营、持续扩展的企业 AI Agent 平台。

如果从客户的角度来讲，它解决的是三件事：

- 把企业知识真正管起来，不再散在文档、群消息和员工经验里
- 把不同业务角色做成不同智能体，而不是全公司只共用一个机器人
- 把问答逻辑、工具调用和业务流程沉淀成可以维护的工作流

## 业务价值

### 1. 一个系统，支持多个业务助手

同一个企业里，可以做多个智能体，例如：

- 制度问答助手
- 销售资料助手
- 交付 SOP 助手
- 客服知识助手
- 培训考试助手

每个智能体都能独立配置名称、欢迎语、推荐问题、知识范围、工作流和模型策略。

### 2. 不只是回答问题，还能承接业务流程

系统支持把回答逻辑做成工作流，不再局限于“检索一下就回答”。

可以组合：

- 知识检索
- AI 生成
- 条件判断
- HTTP 接口
- MCP 服务
- 脚本执行
- 子流程
- 通知动作

这意味着它不仅能做知识问答，也能做流程助手和业务助手。

### 3. 知识库能长期运营

当前知识管理不是粗糙的单目录堆文件，而是按下面的方式管理：

- 知识库
- 分类
- 标签
- 文件

这样做的好处是：

- 后期扩内容不会乱
- 可以按智能体圈定知识范围
- 可以给不同客户、不同部门做内容隔离

## 技术能力

### 后端能力

- FastAPI 提供接口与静态页面入口
- SQLite 维护平台、租户、用户、智能体、日志等核心数据
- 工作流运行时负责执行节点、传递变量和串联调用
- 文档处理模块负责解析上传文件、切片和检索前准备

### 检索与模型能力

- 支持主模型和备用模型
- 支持 Embedding、Rerank
- 支持 Query Rewrite、检索重试、兜底策略
- 支持本地向量目录占位和后续正式数据重建

### 平台运营能力

- 聊天日志
- 请求日志
- 护栏事件
- 评测记录
- 采集运行记录
- 平台和租户分析页

## 版本怎么选

### 企业版适合谁

- 单企业内部使用
- 单客户私有化部署
- 希望快速交付一个企业 AI 控制台

### 服务商版适合谁

- 服务商统一托管多个客户
- SaaS 运营团队
- 需要平台后台和租户后台双层管理

## 当前代码结构

- `backend/`：核心后端逻辑
- `frontend/`：登录、聊天、后台、分析页
- `data/`：平台和租户配置、数据库、检索配置
- `knowledge/`：知识文件目录
- `scripts/`：导出、运维、辅助脚本

## 客户最关心的结果

- 能部署：可以，单机就能启动
- 能配置：可以，大部分品牌和业务配置都在后台完成
- 能扩内容：可以，知识库和智能体都支持持续新增
- 能二开：可以，当前就是源码级交付
