# 二开与接口说明

## 二开先看哪里

- `backend/main.py`：主要接口和页面入口
- `backend/workflow_runtime.py`：工作流执行核心
- `backend/document_processing.py`：文档解析和切片
- `backend/knowledge_assets.py`：知识结构和文件元数据
- `backend/database.py`：数据库读写
- `backend/release_profiles.py`：企业版和服务商版的拆分规则

## 配置文件怎么分

- `data/app_config.json`：平台级业务配置
- `data/model_config.json`：平台级模型配置
- `data/retrieval_config.json`：平台级检索配置
- `data/tool_config.json`：平台级工具和 MCP 配置
- `data/workflow_config.json`：平台级默认工作流
- `data/tenants/<tenant_id>/...`：租户级覆盖配置

## 接口大类

- 平台接口：租户管理、平台日志、平台分析
- 租户接口：品牌配置、模型配置、知识管理、工作流管理、问答测试
- 前台接口：登录、会话、聊天、历史记录
- 运维接口：采集、评测、版本导出

## 再次发版怎么做

```bash
python3 scripts/export_all_releases.py
```

导出后会固定刷新：

- `output/releases/lok_enterprise/`
- `output/releases/lok_service_provider/`
- `output/releases/lok_enterprise_latest.zip`
- `output/releases/lok_service_provider_latest.zip`

## 二开原则

- 通用能力改母版，不要长期改导出包
- 客户差异优先走租户配置，不要硬编码
- 版本差异统一收口在 `backend/release_profiles.py`
